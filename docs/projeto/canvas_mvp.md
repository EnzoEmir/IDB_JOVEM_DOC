# Canvas MVP

## Introdução

O **Canvas MVP (Minimum Viable Product)** é uma ferramenta utilizada para organizar e visualizar, de forma simples, os principais elementos necessários para o desenvolvimento de um produto inicial. O conceito de MVP foi popularizado por Eric Ries (2011) e está relacionado à ideia de construir uma versão básica do produto que permita validar hipóteses com usuários reais, antes de investir muitos recursos em seu desenvolvimento completo.

Nesse sentido, o Canvas MVP ajuda a equipe a alinhar o entendimento sobre o problema que está sendo resolvido, a solução proposta e quais funcionalidades são realmente essenciais. Além disso, a ferramenta permite identificar métricas importantes e os principais perfis de usuários envolvidos no sistema.

Por ser uma abordagem visual e colaborativa, o Canvas facilita a comunicação entre os membros da equipe e contribui para tomadas de decisão mais rápidas. Dessa forma, seu uso está diretamente relacionado a práticas ágeis e ao desenvolvimento incremental, permitindo que o produto evolua com base em feedbacks reais dos usuários (BLANK; DORF, 2012).

---

## Metodologia

A construção do Canvas MVP foi realizada durante uma dinâmica baseada na metodologia **Lean Inception**, proposta por Caroli (2018). Essa abordagem tem como objetivo alinhar a equipe em relação ao produto que será desenvolvido, definindo de forma colaborativa o escopo do MVP.

Durante a atividade, os integrantes do grupo participaram de discussões para identificar os principais problemas enfrentados pelos usuários e propor possíveis soluções. A partir disso, foram levantadas ideias de funcionalidades, que posteriormente foram analisadas e priorizadas de acordo com sua relevância para a primeira versão do sistema.

Além disso, também foram definidas métricas para acompanhar o desempenho do produto, bem como os perfis de usuários que irão utilizá-lo. Esse processo permitiu que diferentes pontos de vista fossem considerados, contribuindo para uma visão mais completa do projeto.

Ao final da dinâmica, foi possível consolidar uma versão única do Canvas MVP, representando o consenso da equipe. Esse artefato servirá como base para a organização do backlog e para o planejamento das próximas etapas do desenvolvimento.
